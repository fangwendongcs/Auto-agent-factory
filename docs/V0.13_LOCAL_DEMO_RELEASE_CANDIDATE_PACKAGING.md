# V0.13 Local Demo / Release Candidate Packaging

Target:

```text
Package the current repo-side workflow control-plane prototype so a new reader can understand and replay it locally.
```

This phase is documentation and local-demo packaging only. It does not change workflow JSON and does not connect to n8n runtime.

## 1. What changed

- README Quick Start now points to the safe local demo path.
- `docs/LOCAL_DEMO_RUNBOOK.md` explains the one-command and step-by-step local replay.
- `npm run demo:local` runs the safe repo-side demo sequence.
- docs index now surfaces the local demo runbook near the top.

## 2. Demo command

```bash
npm run demo:local
```

The demo runs local tests, workflow validation, audit report generation, sign-off review generation, and local review cycle replay.

## 3. Safety boundary

The demo remains:

- local-only
- dev-only
- read-only with respect to n8n runtime
- not production-ready
- not autonomous execution

The only generated files are under `.local-audit/`, which is ignored by Git.

## 4. Release-candidate meaning

V0.13 is a local demo / release-candidate packaging milestone. It means the project can be reviewed and replayed locally, not that it is ready for production use.
