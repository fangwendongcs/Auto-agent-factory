# V0.8f Staging-style Pilot Replay / Audit Export Closeout

Target:

```text
Replay a staging-style read-only run record through the dev-only audit CLI and confirm safe disabled-by-default and explicit dev JSONL behavior.
```

This phase uses sanitized fixture data only. It does not modify n8n runtime, call a provider, read credentials, or enable production storage.

## 1. Replay fixture

Fixture:

```text
examples/audit-replay/v08f-read-only-run-record.json
```

The fixture represents a read-only staging replay record and contains:

- sanitized run id and task id
- approval decision summary
- provider mode and status summary
- timestamps
- no-write safety flags
- redaction metadata

It does not contain:

- full prompt
- provider messages
- provider raw full response
- Authorization header
- API key
- credential
- token
- secret

## 2. CLI default replay

Command:

```bash
npm run audit:dev:replay
```

Expected result:

```json
{
  "ok": true,
  "written": false,
  "reason": "audit_storage_disabled"
}
```

This proves the replay path validates the fixture but does not write unless explicitly enabled.

## 3. Explicit dev-only write replay

Command:

```bash
AUDIT_STORAGE_ENABLED=true \
AUDIT_STORAGE_MODE=dev-jsonl \
AUDIT_STORAGE_PATH=.local-audit/v08f-replay.jsonl \
node scripts/write-dev-audit-record.mjs examples/audit-replay/v08f-read-only-run-record.json
```

Expected result:

```json
{
  "ok": true,
  "written": true,
  "output_path": ".local-audit/v08f-replay.jsonl"
}
```

The output path remains under `.local-audit/`, which is ignored by Git.

## 4. Safety checks

The replay confirms:

- default path writes nothing
- explicit dev-only path writes only under `.local-audit/`
- generated local audit files are ignored by Git
- sanitized records do not include secret-like values
- raw provider request and raw provider response fields are not stored
- no n8n runtime integration is enabled

## 5. Tests

Automated tests cover:

- fixture schema validation
- default CLI replay writes nothing
- explicit dev-only CLI replay writes sanitized JSONL under `.local-audit/`
- written record excludes Authorization, Bearer, provider raw response, and provider request fields

## 6. Boundary

V0.8f does not enable:

- production storage
- n8n runtime auto-write
- real provider calls
- credential reads
- file writes from workflow execution
- shell execution
- Git modification
- external write actions
- production autonomous execution

## 7. Next recommendation

A safe next phase can be:

```text
V0.9 audit review report generation
```

V0.9 should summarize sanitized audit records and produce a human-readable local report without connecting to production workflow writes.
