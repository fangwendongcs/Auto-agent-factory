# V0.8d Dev-only JSONL Audit Storage Prototype

Target:

```text
Implement a disabled-by-default, dev-only JSONL storage prototype for sanitized audit records.
```

This phase implements local utility code only. It does not modify n8n runtime and does not enable production audit persistence.

## 1. What was implemented

Repo-side implementation:

```text
src/utils/auditStorage.js
tests/audit-storage.test.js
```

Supporting update:

```text
src/schema/audit-record.schema.json
.gitignore
```

The storage prototype can append sanitized audit records to JSONL only when explicitly enabled for local development.

## 2. Default behavior

Default behavior is disabled:

```text
AUDIT_STORAGE_ENABLED=false
```

If no enablement flag is provided, the storage utility returns:

```json
{
  "written": false,
  "reason": "audit_storage_disabled"
}
```

No file is created.

## 3. Dev-only enablement

A future local-only run can explicitly enable storage with:

```text
AUDIT_STORAGE_ENABLED=true
AUDIT_STORAGE_MODE=dev-jsonl
AUDIT_STORAGE_PATH=.local-audit/dev-audit-log.jsonl
```

Allowed path:

```text
.local-audit/*.jsonl
```

The prototype rejects paths outside `.local-audit/`.

## 4. Git safety

`.gitignore` includes:

```text
.local-audit/
```

Audit logs generated during local development must not be committed.

## 5. Secret and raw data protection

Before writing, the prototype validates:

- audit record schema
- denied field names
- secret-like string values
- `safety.secrets_stored = false`
- `safety.raw_prompt_stored = false`
- `safety.raw_provider_response_stored = false`

The prototype refuses records containing denied fields such as:

- prompt fields
- provider request fields
- provider raw response fields
- authorization fields
- API key fields
- token / secret / password / credential / cookie fields
- raw headers / raw body fields

## 6. Storage format

Each record is written as one JSON object per line.

When written, the storage metadata is updated to:

```json
{
  "storage": {
    "mode": "dev_jsonl",
    "persisted": true
  }
}
```

The original sanitizer still defaults to:

```json
{
  "storage": {
    "mode": "mock_repo_contract",
    "persisted": false
  }
}
```

## 7. Tests

Tests verify:

- disabled-by-default behavior writes nothing
- explicitly enabled dev JSONL writes under `.local-audit/`
- paths outside `.local-audit/` are rejected
- denied fields and secret-like values are refused

## 8. Boundaries

V0.8d does not enable:

- n8n runtime persistence
- production storage
- database writes
- file writes from workflow execution
- shell execution
- Git modification
- external write actions
- raw prompt storage
- raw provider response storage
- secret storage
