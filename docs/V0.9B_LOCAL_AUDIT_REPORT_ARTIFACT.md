# V0.9b Local Audit Report Artifact Option

Target:

```text
Allow a human-readable audit review report to be saved as a local dev-only Markdown artifact, while keeping stdout-only as the default behavior.
```

This phase extends the V0.9 audit report CLI with an explicit local artifact option. It remains repo-side only and does not connect to n8n runtime.

## 1. Default behavior

By default, the report CLI writes Markdown to stdout only:

```bash
npm run audit:report
```

No report file is written unless artifact writing is explicitly enabled.

## 2. Explicit dev-only artifact mode

To write a local Markdown report artifact:

```bash
AUDIT_REPORT_WRITE_ENABLED=true \
AUDIT_REPORT_OUTPUT_PATH=.local-audit/reports/latest-audit-report.md \
npm run audit:report
```

Allowed output location:

```text
.local-audit/reports/*.md
```

`.local-audit/` is ignored by Git and must not be committed.

## 3. Path restrictions

The artifact writer rejects:

- absolute paths
- path traversal using `..`
- paths outside `.local-audit/reports/`
- non-`.md` files

This keeps the feature local-only and prevents accidental writes into tracked project files.

## 4. Content restrictions

Before writing, the CLI checks the generated report for denied content patterns.

The report artifact must not include:

- provider request payload fields
- unredacted provider response fields
- prompt payload fields
- provider message payload fields
- credential header values
- API keys
- tokens
- passwords
- cookies
- credentials
- secret-like values

The report is generated from sanitized audit records only.

## 5. What this does not enable

V0.9b does not enable:

- n8n runtime integration
- production audit reporting
- database-backed storage
- real provider calls
- credential reads
- workflow-triggered file writes
- shell execution
- Git modification
- external write actions
- production autonomous execution

## 6. Tests

Tests cover:

- default stdout-only behavior does not write a report file
- explicit dev-only environment variables write `.local-audit/reports/*.md`
- paths outside `.local-audit/reports/` are rejected
- non-Markdown artifact paths are rejected
- secret-like report content is refused
- JSONL multi-record report summaries still work

## 7. Next recommendation

A safe next phase can be:

```text
V0.10 audit review checklist / human sign-off workflow design
```

Keep the next phase read-only and human-in-the-loop. Do not connect artifact generation to automatic n8n runtime writes without a separate approval and rollback design.
