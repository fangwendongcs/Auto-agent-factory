# V0.5 Sandbox Manual Setup Checklist

This checklist is the manual stop point before any real provider sandbox call.

The repository is prepared for a read-only provider skeleton, but this checklist must be completed by the user in n8n UI before any real sandbox provider test.

## 1. Current boundary

Current project status:

```text
mock / dry-run / real-readonly stub + read-only provider skeleton
```

This is not production autonomous execution.

The current repository workflow must not:

- write files
- execute shell commands
- modify Git
- call external write APIs
- auto-approve provider output
- expose API keys in Git

## 2. Backup before changing n8n runtime

Before changing anything in n8n UI:

1. Open `[GoalDriven] 02 Agent Task Executor`
2. Export the current workflow
3. Save the backup as:

   ```text
   agent_task_executor-before-v05-provider-sandbox.json
   ```

4. Confirm `[GoalDriven] 01 Master` still calls the intended 02 Executor workflow
5. Do not create a duplicate 02 workflow

## 3. Create sandbox credential in n8n UI

Create a sandbox credential in n8n UI only.

Recommended credential name:

```text
goald-openai-compatible-readonly
```

Rules:

- paste the real API key only into n8n credential UI
- do not paste the key into Codex prompts
- do not paste the key into repository files
- do not paste the key into workflow JSON
- do not paste the key into examples
- do not paste the key into logs

## 4. Configure provider placeholders

Configure these values in n8n runtime or local ignored environment only:

```text
OPENAI_COMPATIBLE_BASE_URL
OPENAI_COMPATIBLE_MODEL
OPENAI_COMPATIBLE_TIMEOUT_MS
```

Do not commit real values.

Acceptable repository placeholders:

```text
{{ $env.OPENAI_COMPATIBLE_BASE_URL }}
{{ $env.OPENAI_COMPATIBLE_MODEL }}
goald-openai-compatible-readonly
```

## 5. Confirm Executor structure

`[GoalDriven] 02 Agent Task Executor` should contain:

- `Real-readonly Provider Selector`
- `Resolve Provider Runtime Config`
- `Real-readonly Safety Check`
- `OpenAI-compatible Provider Request Builder`
- `Provider Response Normalizer`
- `Result Normalizer`
- `Execution Logger`
- `Mock Agent Adapter`
- `Dry-run Provider Adapter`
- `Real-readonly Provider Adapter`

Current verified routes must remain:

```text
mock → Mock Agent Adapter
dry-run → Dry-run Provider Adapter
real-readonly stub → Real-readonly Provider Adapter
provider readiness fallback → Provider Response Normalizer fallback
provider sandbox call → OpenAI-compatible HTTP Request → Provider Response Normalizer
```

## 6. Important implementation note

The repository-side V0.5b workflow is still read-only, but it now contains the HTTP Request node needed for a sandbox provider call.

It resolves provider endpoint and model through a dedicated n8n Set node before entering Code nodes:

```text
OPENAI_COMPATIBLE_BASE_URL
OPENAI_COMPATIBLE_MODEL
```

This avoids relying only on Code node access to runtime environment variables.

The real API key must still live only in the n8n credential:

```text
goald-openai-compatible-readonly
```

Do not treat this as production execution. Provider output remains `needs_review` and cannot trigger writes.

## 8. Manual stop point

Stop here until the user confirms:

- n8n UI is open
- current 02 Executor is backed up
- sandbox credential is created
- real key is stored only in n8n credential UI
- provider base URL is configured outside Git
- provider model is configured outside Git
- user explicitly authorizes one read-only sandbox provider call

## 9. After manual setup

After the manual setup is complete, the next phase is:

```text
V0.5b real provider sandbox call regression
```

That phase should verify:

- provider success path
- missing endpoint fallback
- missing credential fallback
- provider error fallback
- mock regression
- dry-run regression
- real-readonly stub regression
- high-risk approval gate
- invalid payload validation
- secret audit
