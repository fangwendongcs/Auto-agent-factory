# V0.12 Local Review Cycle Replay / End-to-end Documentation

Target:

```text
Make the complete local review cycle reproducible with one command.
```

This phase connects the repo-side dev-only pieces into a single local demonstration flow. It does not modify n8n runtime, does not call providers, and does not enable production approval.

## 1. Replay chain

The local replay performs:

```text
sample audit record
→ audit report
→ signoff review
→ signoff decision
→ ledger append
→ ledger summary
```

## 2. Command

Run:

```bash
npm run audit:cycle:replay
```

The command uses:

```text
examples/audit-replay/v08f-read-only-run-record.json
examples/signoff-decisions/sample_readonly_needs_review_decision.json
```

## 3. Local artifacts

The replay writes only under `.local-audit/`:

```text
.local-audit/reports/v012-*-audit-report.md
.local-audit/signoff/v012-*-signoff-review.md
.local-audit/signoff-ledger/v012-*-ledger.jsonl
```

`.local-audit/` is ignored by Git and must not be committed.

## 4. Output

The command prints:

- replay status JSON to stderr
- ledger summary Markdown to stdout

The ledger summary includes:

- total decision records
- approved / rejected / needs-review counts
- high-risk / write-like / forbidden counts
- review status
- per-record decision summary

## 5. Safety boundaries

V0.12 does not enable:

- n8n runtime integration
- provider calls
- automatic approval
- workflow-triggered file writes
- shell execution
- Git modification
- external write actions
- production autonomous execution

The replay uses sanitized fixtures only and must not contain:

- prompt payloads
- provider chat payloads
- unredacted provider responses
- credential material
- API keys
- provider credentials
- cookies
- passwords
- tokens
- secrets

## 6. Tests

Tests cover:

- full local replay from fixture to ledger summary
- audit report artifact creation
- signoff review artifact creation
- signoff ledger append
- ledger summary generation
- output does not include secret-like values or raw payload field names
