# V0.10 Human Sign-off Review Workflow

Target:

```text
Generate a local human sign-off review package from sanitized audit records before any run moves to a next stage.
```

This phase is repo-side only. It does not connect to n8n runtime, does not call providers, and does not enable real write actions.

## 1. What was implemented

Repo-side implementation:

```text
src/utils/generateHumanSignoffReview.js
scripts/generate-human-signoff-review.mjs
tests/human-signoff-review.test.js
```

Package script:

```bash
npm run audit:signoff
```

## 2. Inputs

The sign-off generator accepts:

- one sanitized audit record JSON file
- one JSONL file containing sanitized audit records

The CLI validates each record before rendering the sign-off package.

## 3. Default output

By default, the CLI writes Markdown to stdout only:

```bash
npm run audit:signoff
```

No file is written unless local artifact writing is explicitly enabled.

## 4. Optional dev-only artifact mode

To write a local sign-off review artifact:

```bash
HUMAN_SIGNOFF_WRITE_ENABLED=true \
HUMAN_SIGNOFF_OUTPUT_PATH=.local-audit/signoff/latest-signoff-review.md \
npm run audit:signoff
```

Allowed output location:

```text
.local-audit/signoff/*.md
```

`.local-audit/` is ignored by Git and must not be committed.

## 5. Review package contents

The Markdown package includes:

- run id / task id
- status
- risk level
- action class
- permission level
- existing approval decision
- provider mode / status
- safety flags
- redaction status
- storage status
- human checklist
- recommended decision
- manual sign-off block

## 6. Recommended decision logic

The local recommendation is intentionally conservative:

| Condition | Recommended decision |
|---|---|
| Sanitized safety checks fail | `reject` |
| Forbidden action | `reject` |
| High-risk action without explicit approval | `needs_review` |
| Write-like action | `needs_review` |
| Safe read-only record | `needs_review` |

A safe read-only record is not auto-approved because this package is not an approval system. It only prepares a human-readable review surface.

## 7. Safety boundaries

V0.10 does not enable:

- n8n runtime integration
- provider calls
- automatic approval
- workflow-triggered file writes
- shell execution
- Git modification
- external write actions
- production autonomous execution

The sign-off package must not contain:

- prompt payloads
- provider message payloads
- unredacted provider responses
- credential header values
- API keys
- provider credentials
- cookies
- passwords
- tokens
- secrets

## 8. Tests

Tests cover:

- stdout-only default behavior
- explicit dev-only artifact writing under `.local-audit/signoff/*.md`
- rejection of paths outside `.local-audit/signoff/`
- rejection of non-Markdown artifact paths
- forbidden action recommendation = `reject`
- high-risk / write-like recommendation = `needs_review`
- safe read-only checklist generation
- secret-like content refusal
- JSONL multi-record rendering

## 9. Next recommendation

```text

```

Keep it repo-side and disabled by default until a separate n8n runtime approval integration is designed and reviewed.
