# V0.8e Dev-only Audit CLI

Target:

```text
Provide a local-only CLI for manually writing sanitized audit records to dev JSONL storage.
```

This phase does not connect audit storage to n8n runtime. It remains disabled by default and is not a production audit system.

## 1. CLI

```bash
node scripts/write-dev-audit-record.mjs <sanitized-audit-record.json>
```

Package script:

```bash
npm run audit:dev:write
```

The package script uses:

```text
examples/sample_sanitized_run_record.json
```

## 2. Default behavior

By default, the CLI validates the record and writes nothing.

Expected output shape:

```json
{
  "ok": true,
  "written": false,
  "reason": "audit_storage_disabled",
  "output_path": ".local-audit/dev-audit-log.jsonl",
  "run_id": "gd_...",
  "task_id": "task_...",
  "storage_mode": "disabled"
}
```

## 3. Explicit dev-only enablement

To write locally during development:

```bash
AUDIT_STORAGE_ENABLED=true \
AUDIT_STORAGE_MODE=dev-jsonl \
AUDIT_STORAGE_PATH=.local-audit/dev-audit-log.jsonl \
node scripts/write-dev-audit-record.mjs examples/sample_sanitized_run_record.json
```

The CLI only supports:

```text
AUDIT_STORAGE_MODE=dev-jsonl
```

The output path must stay under:

```text
.local-audit/
```

## 4. Storage safety

The CLI refuses to write if:

- the input is not valid JSON
- the record fails `audit-record.schema.json`
- the record contains denylisted fields
- the record contains secret-like values
- the output path is outside `.local-audit/`
- the output path is absolute
- the output path traverses with `..`
- the output path does not end with `.jsonl`

## 5. Data that must not be stored

The CLI must not store:

- full prompts
- raw provider request messages
- raw provider full response
- Authorization headers
- API keys
- n8n API keys
- provider credentials
- cookies
- passwords
- secrets

## 6. Git safety

`.local-audit/` is ignored by Git.

Do not commit generated local audit logs.

Verify with:

```bash
git check-ignore -v .local-audit/dev-audit-log.jsonl
```

## 7. Tests

Tests cover:

- CLI default disabled behavior
- CLI explicit dev JSONL write under `.local-audit/`
- path rejection outside `.local-audit/`
- secret-like value rejection

Run:

```bash
npm test
npm run workflow:validate:all
npm run workflow:dry-run
npm run import:check
```

## 8. Boundary

V0.8e does not enable:

- n8n runtime audit persistence
- production storage
- real provider calls
- file writes from workflow execution
- shell execution
- Git modification
- external write actions
- production autonomous execution

## 9. Next recommendation

A safe next phase can be:

```text
V0.8f staging-style pilot replay
```

That phase should use sanitized sample records or manually exported sanitized runtime results. It should not connect the CLI directly to n8n runtime yet.
