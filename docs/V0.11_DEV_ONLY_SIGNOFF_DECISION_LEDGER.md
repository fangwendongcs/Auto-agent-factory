# V0.11 Dev-only Human Sign-off Decision Ledger + Local Review Cycle

Target:

```text
Close the local human review loop with sanitized decision records, a dev-only decision ledger, and a read-only ledger summary report.
```

This phase is repo-side only. It does not connect to n8n runtime, does not call providers, and does not enable production approval or real write actions.

## 1. Local review cycle

The intended local-only chain is:

```text
sanitized audit record
→ audit review report
→ human sign-off review package
→ human decision record
→ dev-only decision ledger
→ ledger summary / review status report
```

## 2. What was implemented

Repo-side implementation:


## 3. Decision record

A sanitized human sign-off decision record includes:

- decision id
- run id / task id
- reviewer summary
- decision: `approved`, `rejected`, or `needs_review`
- reason summary
- source audit record reference
- risk snapshot
- approval snapshot
- storage status
- safety flags
- redaction status

It does not store prompt payloads, provider chat payloads, unredacted provider responses, credential material, or provider secrets.

## 4. Decision validation rules

The validator enforces:

- forbidden action cannot be recorded as `approved`
- unsafe or unredacted run cannot be recorded as `approved`
- high-risk or write-like decision must include an explicit reason summary
- decision record must declare no auto approval
- decision record must declare no side effects
- decision record must declare no secret storage
- denylisted fields are rejected
- secret-like values are rejected

Safe read-only work may be recorded as `approved`, `rejected`, or `needs_review`, but only from explicit human input. The tool never auto-approves a run.

## 5. Decision write CLI

Default behavior:

```bash
npm run audit:signoff:decision
```

The command prints a sanitized decision record to stdout and writes nothing.

Explicit dev-only ledger write:

```bash
SIGNOFF_LEDGER_WRITE_ENABLED=true \
SIGNOFF_LEDGER_MODE=dev-jsonl \
SIGNOFF_LEDGER_PATH=.local-audit/signoff-ledger/dev-signoff-ledger.jsonl \
npm run audit:signoff:decision
```

Allowed output location:

```text
.local-audit/signoff-ledger/*.jsonl
```

`.local-audit/` is ignored by Git and must not be committed.

## 6. Ledger summary CLI

The ledger summary command reads a sanitized decision JSONL fixture and prints a Markdown summary:

```bash
npm run audit:signoff:ledger
```

The summary includes:

- total records
- valid / invalid record counts
- approved / rejected / needs-review counts
- high-risk / write-like / forbidden counts
- review status
- per-record decision summary

It is read-only and writes no files.

## 7. Safety boundaries

V0.11 does not enable:

- n8n runtime integration
- provider calls
- automatic approval
- workflow-triggered file writes
- shell execution
- Git modification
- external write actions
- production autonomous execution

The ledger must not contain:

- prompt payloads
- provider chat payloads
- unredacted provider responses
- credential material
- API keys
- provider credentials
- cookies
- passwords
- tokens
- secrets

## 8. Tests

Tests cover:

- rejection of non-JSONL ledger paths
- forbidden / unsafe run cannot be approved
- high-risk / write-like decision requires explicit reason
